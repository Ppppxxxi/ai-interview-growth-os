# AI 面试成长 OS

面向产品经理求职者的 AI 面试经验沉淀工具。它不是一个普通的模拟面试工具，而是放在 ChatGPT、Claude、Codex 等外部 AI 面试对话之后的“第二层工作台”：把分散的 JD、模拟面试问答、AI 点评和个人复盘，整理成可追溯、可编辑、可复用的回答资产。

## 3 分钟读懂

**用户痛点**

高频面试用户通常会在不同 AI 对话、不同岗位材料和不同复盘文档之间来回切换。一次面试失败后，岗位本身可能不再继续，但其中暴露的问题、可复用的表达框架、被追问的角度，仍然应该服务下一场同岗位多轮或同方向类似岗位面试。

**产品定位**

AI 面试成长 OS 不替代外部 AI 做模拟面试。它更像面试准备的“经验沉淀层”：

- 外部 AI 负责陪练和生成对话。
- 本产品负责导入、解析、复盘、生成优化回答、沉淀回答资产。
- 用户最终得到的是“下一场面试该怎么答”，而不只是一次分数或几条泛泛建议。

**当前 MVP 闭环**

```text
粘贴 JD
  -> 粘贴外部 AI 模拟面试对话
  -> 解析原问题、原回答、追问和点评
  -> 生成结构化复盘
  -> 生成可编辑的优化回答
  -> 用户确认后保存为回答资产
  -> 在同岗位多轮 / 同方向类似岗位中复用
```

当前 Demo 使用“星河智能 AI 产品经理实习生”作为主样例，演示一条完整数据链路：JD + 外部对话 -> 复盘 -> 优化回答 -> 回答资产 -> 复用场景。

## 3 分钟试用路径

1. 进入“岗位工作台”，选择已有样例岗位，或在左侧点击“填入示例岗位”后保存。
2. 点击“填入示例内容”，自动填入 JD 和一段外部 AI 模拟面试对话。
3. 点击“解析并预览”，确认系统识别出的原问题、原回答、追问和点评。
4. 点击“生成复盘与优化回答”，查看复盘详情和优化后的回答。
5. 点击“确认并保存”，将优化回答沉淀为回答资产。
6. 进入“回答资产库”，搜索资产、记录使用反馈、保存回答新版本，或导出 Markdown。

## 核心页面

- **岗位工作台**：默认首页。聚合 JD、外部对话导入、复盘、优化回答、修改依据、确认保存和该岗位关联资产。
- **回答资产库**：核心产出页。支持关键词检索，并按岗位方向、问题类型、能力短板、来源岗位、置信度和是否已使用筛选回答资产。
- **全局成长**：长期视图。展示跨岗位重复短板、资产沉淀和考前优先练习，不在数据不足时强行展示复杂均分图谱。

## v0.1 边界

当前版本是 GitHub-ready 作品集 Demo，重点证明产品闭环和 AI 产品设计判断。

已完成：

- React + TypeScript + Vite 本地 Demo。
- deterministic mock agent，保证无需 API Key 即可运行。
- 外部对话导入到结构化复盘的演示链路。
- 回答资产生成、编辑确认和保存到运行时状态。
- Agent 合约、Roadmap、发布检查和公开仓库风险说明。

暂不做：

- 数据持久化、登录、云同步。
- 真实 LLM API 调用。
- 完整新手引导 wizard。
- 文件上传、浏览器插件、求职 CRM。
- 大规模真实用户数据分析。

## v0.2 Personal MVP 方向

v0.2 的目标是从“作品集 Demo”推进到“个人真实求职可用工具”。第一阶段优先补齐本地使用地基：

- 岗位档案可以在页面中新建。
- 岗位档案可以编辑公司、岗位、方向、阶段和 JD，也可以删除非最后一个岗位。
- 新手可以使用示例岗位和示例面试对话快速跑通完整流程。
- 岗位和回答资产会保存到浏览器 `localStorage`。
- 新建岗位默认不带样例对话，用户可以粘贴自己的 JD 和外部 AI 模拟面试对话。
- 外部面试对话解析结果可以先预览和手动编辑，再用于生成复盘。
- 每次生成复盘都会保存为该岗位的一场面试记录，并进入本地历史记录。
- 可以在岗位工作台切换历史面试记录，查看该场的问答、复盘和关联回答资产。
- 可以删除误导入或测试生成的面试记录，并同步清理关联复盘和回答资产。
- 回答资产可以记录使用结果、使用岗位、关联面试、面试官追问和使用后复盘。
- 回答资产可以保存新版本、查看历史回答，并将历史版本恢复为当前推荐回答。
- 可以导出单场面试复盘、岗位考前准备包和回答资产合集为 Markdown。
- GitHub 仓库仍只包含代码和 mock 示例数据，真实 JD、真实面试对话和个人复盘只保存在本地浏览器。

后续再逐步增加真实 LLM 接入、公开演示截图和部署说明。

## AI Agent 设计

当前本地 Demo 使用 deterministic mock agent，真实上线版本会在保持输入输出合约稳定的前提下替换为 LLM 调用：

1. `importParser`：外部对话解析。
2. `jdAnalyst`：JD 画像分析。
3. `reviewEvaluator`：结构化复盘生成。
4. `answerAssetGenerator`：优化回答与回答资产生成。
5. `growthPlanner`：跨场次短板识别。

详细合约见 [docs/agents/agent-contracts.md](docs/agents/agent-contracts.md)。

## 本地运行


```powershell

npm.cmd run build
```

PowerShell 环境建议使用 `npm.cmd`，避免系统执行策略拦截 `npm.ps1`。
