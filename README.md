# AI 面试成长 OS

面向产品经理求职者的 AI 面试训练与复盘系统。MVP 先聚焦 AI 产品经理面试准备，目标是把分散在不同 AI 对话、岗位档案和复盘文档中的面试经历，沉淀为可复用的能力图谱、回答资产和下一轮训练计划。

## 项目背景

很多求职者已经会用 ChatGPT、Codex 或其他 AI 工具进行模拟面试，但实际使用中会遇到几个问题：

- 每个岗位都需要单独开一个对话，历史经验难以复用。
- JD 分析、问题、回答、追问、点评和文档过多，信息难以管理。
- 复盘结果往往停留在单次对话中，无法沉淀为长期能力资产。
- 下一场面试前，很难从过往所有面试经历中提取可迁移问题和经验。

本项目尝试解决的核心问题是：**如何把每一次面试经历转化为下一次面试可复用的准备资产。**

## 产品定位

AI 面试成长 OS 不是普通模拟面试聊天机器人，而是一个长期面试成长系统。

核心结构：

- **成长首页**：展示全局能力图谱、高频短板、回答资产和下一步训练任务。
- **岗位 / 面试档案**：承接每个岗位的 JD、经历匹配、模拟面试和复盘记录。
- **面试复盘**：将内置模拟或外部 AI 对话转化为结构化复盘报告。
- **回答资产库**：沉淀可复用回答、适用场景和关联经历。
- **训练任务**：基于多次复盘生成用户面试能力训练计划。

## MVP 功能

- 个人经历库：结构化记录实习、项目、研究、竞赛等经历。
- 岗位档案：粘贴 JD，生成岗位画像和准备状态。
- 经历匹配：根据 JD 推荐适合讲述的个人经历。
- 内置模拟面试：基于 JD 和经历库生成问题、追问和点评。
- 外部对话导入：粘贴 ChatGPT / Codex 面试对话，自动抽取问题、回答和反馈。
- 结构化复盘：按岗位理解、产品分析、AI 产品理解、数据指标、项目表达、沟通结构化等维度评分。
- 成长图谱：跨多场面试识别重复短板和能力趋势。
- 回答资产：保存原回答、优化回答、适用场景和关联经历。
- 训练计划：生成用户下一轮面试能力训练任务。


