# AI 面试成长 OS

面向产品经理求职者的 AI 面试经验沉淀工具。它不是一个普通的模拟面试工具，而是放在 ChatGPT、Claude、Codex 等外部 AI 面试对话之后的“第二层工作台”：把分散的 JD、模拟面试问答、AI 点评和个人复盘，整理成可追溯、可编辑、可复用的回答资产。

## 3 分钟读懂

**用户痛点**

高频面试用户通常会在不同 AI 对话、不同岗位材料和不同复盘文档之间来回切换。一次面试失败后，岗位本身可能不再继续，但其中暴露的问题、可复用的表达框架、被追问的角度，仍然应该服务下一场同岗位多轮或同方向类似岗位面试。

**产品定位**

AI 面试成长 OS 不替代外部 AI 做模拟面试。它更像面试准备的“经验沉淀层”：

- 外部 AI 负责陪练和生成对话。
- 本产品负责导入、解析、复盘、生成优化回答、沉淀回答资产。
- 用户最终得到的是“下一场面试该怎么答”，而不只是一次分数或几条泛泛建议。

**当前 MVP 闭环**

```text
导入一整份面试材料
  -> 生成可确认草稿
  -> 用户编辑问题、原回答、问题点和优化回答
  -> 勾选需要沉淀的回答资产
  -> 保存为面试记录和回答资产
  -> 在同岗位多轮 / 同方向类似岗位中复用
```

当前版本提供“真实使用”和“示例体验”两个入口。示例体验使用“星河智能 AI 产品经理实习生”作为主样例；主工作台新增整份材料导入入口，支持把 AI 模拟面试对话、真实面试回忆、复盘笔记或混合材料整理成可确认草稿。

## 3 分钟试用路径

1. 首次进入时选择“开始准备我的面试”，或点击“使用示例体验”快速看完整链路。
2. 在聚焦流程页中粘贴一整份面试材料，或点击“填入示例”查看格式。
3. 点击“解析材料”，检查系统生成的问题、原回答、具体问题和修改建议。
4. 点击“生成下次可用回答”，逐条编辑并勾选需要保存的回答。
5. 点击“保存到我的回答库”，将本次材料沉淀到当前岗位。
6. 进入“我的回答库”，搜索回答、记录使用反馈、保存回答新版本，或导出 Markdown。

## 核心页面

- **开始新面试**：聚焦流程。粘贴整份面试材料，检查可确认草稿，保存下次可用回答。
- **工作台**：沉淀管理。查看历史岗位、面试记录、已保存回答、复盘结论和下一步练习。
- **我的回答库**：核心产出页。支持关键词检索，并按岗位方向、问题类型、能力短板、来源岗位、置信度和是否已使用筛选回答。
- **面试复盘**：长期视图。展示跨岗位重复短板、回答沉淀和考前优先练习，不在数据不足时强行展示复杂均分图谱。

## v0.1 边界

当前版本是公开可运行的产品原型，重点验证产品闭环和 AI 产品设计判断。

已完成：

- React + TypeScript + Vite 本地应用。
- deterministic mock agent，保证无需 API Key 即可运行。
- 外部对话导入到结构化复盘的演示链路。
- 回答资产生成、编辑确认和保存到运行时状态。
- Agent 合约、Roadmap、发布检查和公开仓库风险说明。

暂不做：

- 数据持久化、登录、云同步。
- 真实 LLM API 调用。
- 完整新手引导 wizard。
- 文件上传、浏览器插件、求职 CRM。
- 大规模真实用户数据分析。

## v0.2 Personal MVP 方向

v0.2 的目标是从“可运行产品原型”推进到“个人真实求职可用工具”。第一阶段优先补齐本地使用地基：

- 岗位档案可以在页面中新建。
- 岗位档案可以编辑公司、岗位、方向、阶段和 JD，也可以删除非最后一个岗位。
- 新手可以使用示例岗位和示例面试对话快速跑通完整流程。
- 岗位和回答资产会保存到浏览器 `localStorage`。
- 新建岗位默认不带样例对话，用户可以粘贴自己的 JD 和外部 AI 模拟面试对话。
- 外部面试对话解析结果可以先预览和手动编辑，再用于生成复盘。
- 每次生成复盘都会保存为该岗位的一场面试记录，并进入本地历史记录。
- 可以在岗位工作台切换历史面试记录，查看该场的问答、复盘和关联回答资产。
- 可以删除误导入或测试生成的面试记录，并同步清理关联复盘和回答资产。
- 回答资产可以记录使用结果、使用岗位、关联面试、面试官追问和使用后复盘。
- 回答资产可以保存新版本、查看历史回答，并将历史版本恢复为当前推荐回答。
- 可以导出单场面试复盘、岗位考前准备包和回答资产合集为 Markdown。
- GitHub 仓库仍只包含代码和 mock 示例数据，真实 JD、真实面试对话和个人复盘只保存在本地浏览器。

后续再逐步增加真实 LLM 接入、公开演示截图和部署说明。

## v0.2.1 真实新用户体验修正

v0.2.1 聚焦首次使用体验，而不是继续增加新能力：

- 首次进入先展示价值主张和两个入口：开始真实使用 / 使用示例体验。
- 示例数据加明显标识，不再默认伪装成用户自己的历史数据。
- 示例回答资产默认是“待实战验证”，只有用户记录反馈后才显示已使用。
- 空白岗位下不再展示完整草稿复盘，只提示用户粘贴 JD 和外部面试对话。
- 增加“恢复示例”和“清空本地数据”，让用户知道数据保存在本地浏览器。
- 导航文案改为更贴近用户心智的“准备本场面试 / 可复用回答 / 复盘总结”。

## v0.3 整份面试材料导入

v0.3 开始处理真实使用中更常见的输入形态：用户手里不一定有标准的“JD + 面试官/候选人/点评”格式，而可能是一整段面试回忆、复盘笔记和外部 AI 对话混在一起。

已完成：

- 新增“导入一整份面试材料”主入口。
- 支持自动识别 AI 模拟面试对话、真实面试回忆、复盘笔记和混合材料。
- 新增 `materialImportParser` deterministic mock agent，生成可确认草稿。
- 草稿支持编辑问题类型、原问题、原回答、具体问题、修改建议和下次可用回答。
- 用户可勾选哪些条目保存为回答资产。
- 保存后自动进入当前岗位的面试记录、复盘详情和回答资产库。

仍不做：

- 不接真实 LLM。
- 不做文件上传。
- 不做云端数据持久化。
- 不自动把低置信条目保存为回答资产。

## v0.3.1 真实使用校准

v0.3.1 聚焦真实使用时的首屏理解和流程连续性，不扩展 AI 能力：

- 首页文案改为“粘贴材料 -> 检查草稿 -> 保存回答”，与工作台主流程一致。
- 顶部导航改为更贴近任务的表达，减少内部术语。
- 工作台首屏只保留“导入一整份面试材料”主任务。
- 标准 JD + 对话导入降级为辅助入口，避免两个入口并列。
- 未生成前右侧只展示下一步提示，不再展示 0 场记录、待确认、待生成等负反馈。
- 新建岗位和岗位编辑默认折叠，降低首次进入时的表单压力。
- 生成后再展示面试记录、回答、岗位准备和训练建议。

## v0.3.2 聚焦流程与工作台分流

v0.3.2 不再把首次使用和长期管理混在同一个工作台里，而是按用户状态分流：

- 无本地真实数据：默认进入聚焦流程页。
- 有历史岗位、面试记录或回答：默认进入工作台。
- 老用户可从顶部导航点击“开始新面试”，进入聚焦流程。

聚焦流程只负责一次任务：

```text
粘贴 JD、面试记录或模拟面试对话
  -> 解析材料
  -> 确认识别结果
  -> 生成下次可用回答
  -> 保存到我的回答库
```

工作台只负责沉淀管理：历史岗位、面试记录、已保存回答、复盘总结和下次训练题。

## AI Agent 设计

当前本地版本使用 deterministic mock agent，真实上线版本会在保持输入输出合约稳定的前提下替换为 LLM 调用：

1. `materialImportParser`：整份面试材料解析为可确认草稿。
2. `importParser`：标准外部对话解析。
3. `jdAnalyst`：JD 画像分析。
4. `reviewEvaluator`：结构化复盘生成。
5. `answerAssetGenerator`：优化回答与回答资产生成。
6. `growthPlanner`：跨场次短板识别。

详细合约见 [docs/agents/agent-contracts.md](docs/agents/agent-contracts.md)。

## 本地运行

```powershell
npm.cmd install
npm.cmd run dev
```

```powershell
npm.cmd test
npm.cmd run build
```

PowerShell 环境建议使用 `npm.cmd`，避免系统执行策略拦截 `npm.ps1`。
