# v0.3 Release Notes

## 版本定位

v0.3 是“整份面试材料导入”版本。它解决 v0.2 中输入形态过窄的问题：真实用户往往手里是一整段 AI 模拟面试对话、真实面试回忆、复盘笔记和岗位信息混合在一起，而不是已经整理好的标准问答格式。

本版本仍然是本地 deterministic mock agent，不接真实 LLM。目标是先验证交互闭环：

```text
导入一整份面试材料
  -> 生成可确认草稿
  -> 用户编辑和勾选
  -> 保存为面试记录和回答资产
  -> 后续在同岗位多轮 / 同方向类似岗位中复用
```

## 本版本改了什么

- 新增“导入一整份面试材料”主入口。
- 支持选择材料类型：自动识别、AI 模拟面试对话、真实面试回忆、复盘笔记、混合材料。
- 新增 `materialImportParser` mock agent：
  - 识别公司、岗位、方向和 JD 上下文。
  - 从材料中提取 AB 实验、评测口径、公司业务理解、自我介绍贴岗、AI Agent 链路拆解等高频条目。
  - 生成问题、原回答或材料片段、具体问题、修改建议和下次可用回答。
  - 标记置信度和缺失信息提示。
- 新增可确认草稿 UI：
  - 用户可以逐条编辑问题类型、原问题、原回答、具体问题、修改建议和优化回答。
  - 用户可以勾选哪些条目保存为回答资产。
- 新增 `materialImportDraft` workflow：
  - 把草稿保存为当前岗位的一场面试记录。
  - 生成对应复盘报告。
  - 将勾选条目保存为回答资产。
- 保留原有“JD + 外部 AI 模拟面试对话”快速入口，作为标准格式导入方式。

## 当前边界

- 不接真实 LLM。
- 不做文件上传。
- 不做云端数据持久化。
- 不自动保存低置信条目为回答资产。
- 不解决复杂长材料的深度语义理解，后续由 LLM 版本验证。
