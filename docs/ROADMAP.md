# Roadmap

## v0.1 GitHub-ready Demo

目标：让面试官 3 分钟内理解产品定位、核心闭环和 AI Agent 设计。

已完成：

- 三个主视图：岗位工作台、回答资产库、全局成长。
- 一条完整样例链路：JD + 外部 AI 面试对话 -> 复盘 -> 优化回答 -> 回答资产 -> 复用场景。
- deterministic mock agent，保证本地稳定运行。
- 回答资产确认保存、继续编辑、暂不保存。
- Agent 合约、公开仓库风险说明和发布检查清单。

## v0.2 个人真实使用版

目标：让单个用户可以长期使用，而不只是看 Demo。

已开始：

- 使用 `localStorage` 保存岗位档案和回答资产。
- 支持在岗位工作台中新建岗位档案。
- 支持保存当前岗位 JD 到本地岗位档案。
- 支持对外部面试对话解析结果进行预览和手动编辑，再生成复盘。
- 支持同一岗位保存多场面试记录和对应复盘。
- 支持切换历史面试记录，并查看该场问答、复盘和关联回答资产。
- 支持删除面试记录，并同步清理关联复盘和回答资产。
- 支持按关键词、来源岗位、问题类型、能力短板、置信度和使用状态检索回答资产。
- 支持记录回答资产的使用结果、使用场景、面试官追问和使用后复盘。
- 支持导出单场面试复盘、岗位考前准备包和回答资产合集为 Markdown。

计划：

- 支持编辑和删除岗位档案。
- 支持回答资产版本管理，保留历史回答和当前推荐版本。
- 增加基础新手示例和空状态引导。

仍不做：

- 登录和云同步。
- 多用户协作。
- 大规模统计分析。

## v0.3 LLM 接入实验版

目标：验证真实 LLM 是否明显提升解析、复盘和回答生成质量。

计划：

- 先接入 `importParser` 和 `answerAssetGenerator`。
- 增加 JSON schema 校验和失败兜底。
- 增加 prompt 版本管理。
- 增加用户确认点和低置信度提示。
- 支持用户选择是否发送完整经历材料。

风险：

- 模型可能编造经历或指标。
- 解析复杂对话时可能丢失上下文。
- prompt 成本和延迟会影响真实使用体验。

## v0.4 可信工作流版

目标：让用户愿意把它作为真实面试准备工具。

计划：

- 每条回答资产显示来源证据和适用边界。
- 增加“采纳 / 修改 / 放弃”记录。
- 增加复盘质量反馈，反向优化 prompt。
- 增加本地脱敏和隐私提示。
- 增加导出 Markdown / PDF 复盘文档。

## v1.0 真实可用工具

目标：支持个人持续求职周期。

计划：

- 登录、云同步和多设备访问。
- 岗位看板和面试日程。
- 简历经历库和回答资产联动。
- 浏览器插件或剪贴板导入。
- 可配置模型供应商和隐私策略。

## 暂不扩张的方向

- 不做大而全求职 CRM。
- 不做自动投递。
- 不做薪资谈判或职业规划。
- 不把能力图谱作为第一阶段核心卖点。
- 不把跨岗位复用过度泛化，优先证明同岗位多轮和同方向类似岗位复用。
