# 产品案例级 UI 强化版 V1 设计规格

## 目标

把当前功能原型升级为可展示、可试用的高保真业务工具 Demo。保持“AI 面试成长 OS”的长期定位，但第一屏聚焦一个真实闭环：导入 JD 和外部 AI 面试对话，生成复盘与优化回答，保存为回答资产，并在同岗位多轮或同方向岗位中复用。

## 设计原则

- 像真实工具，不像静态说明页。
- 用户先操作，再看结果；不要只展示样例卡片。
- 回答资产是核心产出，评分和图谱是辅助。
- UI 文案只对用户说话，技术实现和 mock agent 说明留在 README 与文档中。

## 页面结构

### 岗位工作台

采用三栏工作台布局：

- 左栏：岗位列表、当前岗位状态、准备阶段。
- 中栏：主流程，包含 JD 输入、面试对话输入、生成按钮、生成中状态、复盘结果、原回答与优化回答对比。
- 右栏：本岗位回答资产、考前优先训练、保存状态。

主 CTA：`生成复盘与优化回答`。

交互状态：

- `idle`：显示输入区和运行示例提示。
- `generating`：显示分析步骤，包括“识别问题”“定位短板”“生成优化回答”。
- `reviewReady`：展示复盘和回答对比。
- `assetSaved`：右侧资产卡显示已保存状态。

### 回答资产库

改为知识库式布局：

- 顶部筛选栏：岗位方向、问题类型、能力短板、置信度。
- 左侧：回答资产列表，展示问题类型、适用范围、状态。
- 右侧：选中资产详情，展示原问题、原回答、具体问题、优化回答、适用类似问题和使用建议。

### 全局成长

保持轻量：展示重复短板、已沉淀回答、适用岗位和考前训练。不要引入复杂图表。

## 测试要求

- 新增纯 TypeScript 工作流模块，测试生成流程和保存状态。
- 保持 `npm.cmd test` 和 `npm.cmd run build` 通过。

## 不做

- 不接真实 LLM。
- 不引入后端和数据库。
- 不做完整用户登录、文件上传、真实编辑持久化。
